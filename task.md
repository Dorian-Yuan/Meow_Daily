# UI/UX 升级与优化任务列表 (UI/UX Upgrade Tasks)

## 目标
- **UX**: 保留现有功能，优化层级与逻辑（首页：仪表盘/快捷录入；记录：历史流水/CRUD；我的：档案/设置）。确保符合规范的按钮命名（日常、饮食、体重、就诊），以及主「保存并同步」按钮的存在。
- **UI**: 提升高级感与活力，严格符合规范（8pt网格，品牌蓝#4066E0，岁岁橘#F28C38，纯扁平极简线条，极致圆润治愈系），并加入适当动效。卡片采用纯白底色、12-16pt圆角、轻软投影（#00000010, 模糊8pt, 偏移0 2pt）、16pt内边距。字体排版严格遵循字号与对齐规范。

## 原子化任务列表
- [ ] 1. 提交当前工作区所有更改，建立安全回退点（已在分支 ui-ux-upgrade）。
- [ ] 2. 审查并更新 `css/variables.css`：植入规定的核心色彩系统、中性色系、8pt网格间距变量、标准化字号与字体族。
- [ ] 3. 审查并更新 `css/main.css` 与 `css/components.css`：重构基础样式与组件，落实极致圆润风格（圆角≥4pt）、卡片标准阴影与内边距、流式布局和排版对齐要求。
- [ ] 4. 增加微交互动效：为按钮点击（缩放/变色）、卡片悬浮、Tabbar 切换添加自然流畅的过渡动画，提升活力与高级感。
- [ ] 5. 优化 `index.html` DOM 结构：按照“首页、记录、我的”重新梳理视图层级，统一按钮命名为“日常”、“饮食”、“体重”、“就诊”，加入统一的主同步按钮。
- [ ] 6. 适配与调整 JS 逻辑 (`js/modules/ui.js`, `js/app.js`)：调整事件绑定与视图切换逻辑以适应新结构，确保不删除现有功能，无缝继承历史状态。
- [ ] 7. 进行最终视觉与功能检查，确认符合 Spec 和 MEMO 所有的 UI/UX 要求。
