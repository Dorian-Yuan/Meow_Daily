# Meow_Daily V4 开发计划与任务打磨

## 1. 动态提醒设置系统 (System Settings)
- [x] **数据模型支持**：改造 `db.json` 中的 `settings.reminder_cycles` 结构，支持用户自定义增删改。
- [x] **UI 交互**：在“系统设置”页面新增“提醒事项管理”模块，允许用户添加新提醒（如“内外同驱”：30天）、修改或删除现有提醒。
- [x] **后端 Action 适配**：更新 GitHub Actions 的 `daily_job.js` 脚本，使其动态读取用户设置的提醒列表，并在到达提醒日期时通过 ChatAnywhere 生成文案并通过 Bark 发送推送。

## 2. 首页提醒引擎重构 (Home Reminders)
- [x] **纪念日提醒**：判断今天是否为猫咪的生日或到家纪念日，如果是，则在提醒面板中置顶显示特殊庆祝卡片（如“今天是岁岁3岁生日！🎂”）。
- [x] **倒计时与完成状态**：重新设计倒计时逻辑：
    - 显示距离下次提醒**还有多少天**。
    - 如果今天已经完成了该事项，则显示**“今天已完成 ✅”**。
    - 如果超期，显示**“已逾期 X 天 ⚠️”**。

## 3. 图标异常排查与修复 (Meow IP)
- [x] **问题定位**：目前的 `meow-ip.png` 体积过大（达到 10MB），导致 Web 端和 PWA 加载异常或无法显示。
- [x] **修复方案**：对图标进行压缩和尺寸裁剪（建议 512x512，小于 200KB），以符合 PWA Manifest 和网页优化的标准。
