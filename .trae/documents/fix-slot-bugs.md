# 老虎机 Bug 修复计划

## Bug 清单

### Bug 1（严重）：视觉与逻辑不一致
**现象**：玩家看到的符号和实际判定的符号不一致，导致"莫名其妙得分"或"明明三连却没得分"

**根因**：
- `spin()` 生成 23 个符号（20随机 + 3结果），结果在最后 3 个位置
- CSS `slotSpin` 动画只是 `translateY(-70px)` 循环，并不是真正滚动到结果位置
- 停止时 `spinning` 类移除，strip 回到 `translateY(0)`，显示的是**前 3 个随机符号**，不是结果
- `checkWin(results)` 判定的是**后 3 个结果符号**
- 视觉和逻辑完全脱节！

### Bug 2：中线两连判定太宽松
**现象**：中线两连概率 ~28%，太容易得分

**根因**：`midRow[0]===midRow[1] || midRow[1]===midRow[2]` 只要有任意两个相邻相同就算，6 种符号下概率偏高

### Bug 3：winMsg 被覆盖
**现象**：多线同时中奖时只显示最后一条消息

**根因**：上线/下线三连时 `winMsg = ...` 直接覆盖了中线消息

## 修复方案

### 1. 重写旋转动画逻辑
- 每个转轮生成足够多的符号（如 15 个随机 + 3 个结果 = 18 个）
- 旋转时用 `requestAnimationFrame` 驱动 `translateY` 向下滚动
- 停止时用 CSS `transition` 平滑过渡到结果位置（最后 3 个符号可见）
- 确保视觉显示的符号与 `results` 完全一致

### 2. 调整中奖规则
- 只保留三连（上线/中线/下线），移除两连
- 中线三连 +50🥫，上线/下线三连 +30🥫
- 新增：对角线三连 +40🥫（左上→右下，右上→左下）
- 新增：全满贯（3 线三连）额外 +50🥫 奖励

### 3. 修复 winMsg
- 用数组收集所有中奖消息，用换行拼接显示
