# **Meow Phone 扩展内容 — GitHub 调研报告1**

## **一、现有项目分析**

### **项目技术栈**

* **纯 Vanilla JS（ES Modules），无框架**
* **CSS 变量体系，"暖奶茶"配色（背景 #FBF8F4，主色 #4066E0）**
* **PWA（Service Worker + manifest.json）**
* **移动优先响应式设计，底部三栏导航（首页/记录/我的）**
* **字体：Nunito + Noto Sans SC**

### **meow\_phone 现状**

**隐藏彩蛋系统，点击头像 5 次（2 秒内）触发。弹出手机模拟界面，已包含 8 个 mini-app：**

* **Cat Sweep（扫雷）、Pixel Art（像素画）、Yarn Ball（毛线球物理玩具）**
* **Cat Fortune（猫咪占卜）、Cat Memory（记忆翻牌）、Whack Mouse（打地鼠）**
* **Meow Piano（猫咪钢琴）、Meow Timer（计时器）**

**每个 app 对应独立 JS 文件（js/modules/games/），使用 emoji UI + CSS 动画，通过 store.js 持久化设置。**

## **二、推荐新增内容（按适配难度排序）**

### **★★★★★ 极易集成（单文件/零依赖，直接可用）**

#### **1. oneko.js — 桌面追光猫**

* **仓库：<https://github.com/adryd325/oneko.js>**
* **Stars：\~1,200 | 协议：MIT**
* **技术：纯 JS 单文件，零依赖**
* **内容：经典像素猫跟随光标/触摸点移动，有坐、跑、睡、挠等多种动画状态，基于 sprite sheet**
* **适配方式：直接作为 meow\_phone 的"桌面宠物"功能。在手机界面中放一只像素猫跟手指走，用户可以"养"它。只需引入一个 JS 文件 + 一张 sprite 图。可以换自定义猫皮肤**
* **改造量：几乎为零。换色调、调大小即可**

#### **2. ASCII 猫艺术生成器**

* **仓库：<https://github.com/M-Ziraki/cat-ascii-art-generator>**
* **Stars：小型项目 | 协议：MIT**
* **技术：纯 HTML/CSS/JS 单文件**
* **内容：随机生成 ASCII 猫表情（如 =.= 等变体），带复制到剪贴板功能**
* **适配方式：作为 meow\_phone 的"猫咪表情包"或"猫语键盘"。一键生成可爱猫颜文字，长按复制**
* **改造量：极小。调整样式匹配奶茶色系即可**

#### **3. Cat Facts API（无需 clone，直接调用）**

* **API 地址：<https://catfact.ninja/fact>**
* **技术：单次 fetch() 调用返回 JSON**
* **内容：随机猫咪冷知识**
* **适配方式：在 meow\_phone 中加一个"每日猫知识"卡片，每次打开随机刷新一条。可配合现有的 Cat Fortune 做成"猫咪百科"入口**
* **改造量：10 行 JS 代码**

#### **4. CATAAS — Cat as a Service（无需 clone）**

* **API 地址：<https://cataas.com/cat/says/meow>**
* **技术：纯 URL 拼接，返回图片**
* **内容：随机猫咪图片，支持在图片上叠加自定义文字**
* **适配方式：做成"猫咪表情包生成器"，用户输入文字 → 叠加到随机猫图上，生成专属表情包**
* **改造量：20-30 行代码，一个输入框 + 一个 img 标签**

### **★★★★ 易于集成（需少量改造）**

#### **5. chat-bubble — 聊天气泡引擎**

* **仓库：<https://github.com/dmitrizzle/chat-bubble>**
* **Stars：\~600 | 协议：MIT**
* **技术：纯 JS（ES5），零依赖**
* **内容：JSON 驱动的聊天界面，支持分支对话、打字指示器、图片消息**
* **适配方式：做成"和猫咪聊天"功能。预设多组猫咪对话脚本（JSON 格式），用户选择话题后猫咪用"喵语"回复。气泡换成猫爪图标 + 奶茶配色**
* **改造量：引入 JS/CSS 两个文件，编写对话 JSON，调色**

#### **6. TinyMusic — 迷你音乐合成器**

* **仓库：<https://github.com/kevincennis/TinyMusic>**
* **Stars：\~360 | 协议：MIT**
* **技术：纯 JS，Web Audio API，零依赖**
* **内容：用简单字符串定义音符（如 "C4 q"），程序化播放旋律**
* **适配方式：升级现有 Meow Piano。目前的钢琴可能是简单音效，用 TinyMusic 可以做成"猫咪作曲家"——用户编排音符序列，一键播放猫咪旋律。或者做成"猫咪音乐盒"，预设几首猫主题小曲**
* **改造量：引入单文件，编写几段预设旋律**

#### **7. CSS 猫咪加载动画**

* **仓库：<https://github.com/robin-ivi/cat-loader>**
* **Stars：\~5 | 协议：MIT**
* **技术：纯 CSS 动画，零 JS**
* **内容：可爱的像素猫加载动画**
* **适配方式：不是独立 app，而是作为 meow\_phone 各 app 的加载过渡动画。提升整体体验的润色项**
* **改造量：复制 CSS keyframes，调色**

#### **8. 真实猫叫声音频数据集**

* **仓库：<https://github.com/haydenroche5/meow_dataset>**
* **Stars：\~8**
* **技术：WAV 音频文件集**
* **内容：真实猫叫录音，按猫咪个体和场景分类命名**
* **适配方式：做成"猫语翻译器"或"猫咪心情声音"——选择一种猫咪情绪（开心/饿了/撒娇），播放对应真实猫叫声。也可以作为 Meow Piano 的音色素材**
* **改造量：挑选几个 WAV 文件，用 Web Audio API 播放**

## **四、优先级建议**

**考虑到现有 meow\_phone 已有 8 个 app，建议新增内容要有差异化。按"改造成本低 + 趣味性高 + 与现有功能不重叠"排序：**

**第一梯队（建议优先做）：**

1. **oneko.js 桌面追光猫 — 不是 app 而是"手机桌面"本身的增强，像素猫在 app 网格间跑来跑去**
2. **猫语翻译器 — 独特的互动功能，和现有 8 个 app 完全不同类**
3. **猫咪表情包生成器（CATAAS API）— 实用且有分享价值**

**第二梯队（锦上添花）：**

1. **猫咪贪吃蛇 — 新游戏品类，与现有 8 个不重叠**
2. **每日猫知识（catfact.ninja）— 轻量内容填充**
3. **ASCII 猫颜文字 — 小而美，复制分享**

**第三梯队（如有余力）：**

1. **用 TinyMusic 升级 Meow Piano**
2. **用 chat-bubble 做"和猫咪对话"**
3. **CSS 猫动画润色过渡效果**

## Meow Phone UI 风格特征

* **全屏手机模拟器**：竖屏布局，状态栏 + 内容区 + Dock栏

* **Emoji 图标**：28px emoji 在 56×56 圆角方格内（border-radius: 16px）

* **4列网格**：应用图标排列

* **暖色调猫咪主题**：CSS 变量驱动，支持深色模式

* **纯 Vanilla JS**：无框架依赖，游戏模块导出 `createXxxApp(container, prefs)` 工厂函数

* **容器样式统一**：`flex:1; display:flex; flex-direction:column; height:100%; overflow:hidden`

## 新应用集成标准

1. **轻量**：纯 Vanilla JS/HTML/CSS，无框架依赖
2. **易适配**：代码量小，容易改为猫咪主题
3. **有趣**：玩法吸引人，适合碎片时间
4. **适配竖屏**：在手机尺寸容器内运行良好
5. **不重复**：与现有 8 款应用功能不重叠

## 不推荐的项目类型及理由

| 类型              | 理由                                        |
| --------------- | ----------------------------------------- |
| 虚拟宠物/Tamagotchi | 代码量大、需要持久化状态、与现有"猫咪占卜"的每日缓存机制冲突，且需要长期维护状态 |
| Wordle/猜词游戏     | 需要大量中文词库、输入法交互复杂、竖屏体验差                    |
| 贪吃蛇/Flappy Bird | 需要持续操控，与 Meow Phone 的"碎片时间"定位不太匹配         |
| 复杂 Canvas 游戏    | 代码量大、适配成本高、与 Meow Phone 的轻量定位不符           |

## 实施步骤（以"喵喵合并 2048"为例）

1. 在 `js/modules/games/` 下创建 `cat_2048.js`，导出 `createCat2048App(container, prefs)`
2. 在 `meow_phone.js` 顶部 import 新模块
3. 在 `APP_REGISTRY` 中添加 `{ id: 'cat_2048', name: '喵喵合并', icon: '🔢', description: '数字合并挑战', launch: launchCat2048 }`
4. 编写 `launchCat2048()` 启动函数（遵循统一模式）
5. 在 `launchSettings()` 中添加难度设置（4×4 / 5×5）
6. 在 `components.css` 中添加 `.cat-2048-app-container` 样式
7. 在 `db.json` 和 `store.js` 的 `game_prefs` 中添加默认配置
8. 版本号 +0.0.1
