# V3.1.5 像素画板增强计划

## 需求概述

1. **网格线增强**：横向与纵向最中间的那根线、四周最外侧的框线颜色适当加深
2. **透明色支持**：默认打开画板时像素为透明色，橡皮擦擦除后变为透明色，导出 PNG 时支持透明

---

## 一、网格线增强

### 现状分析

当前网格线通过 CSS Grid `gap: 1px` + 容器背景色 `var(--color-divider)` 实现，所有网格线颜色一致。

### 实现方案

放弃 CSS Grid `gap` 方案，改用 **Canvas 绘制网格**（仅用于显示），这样可以精确控制每条线的颜色。

但考虑到当前整个画板交互基于 DOM 元素（div.pixel），切换到 Canvas 显示会大幅重构。更合理的方案是 **保留 CSS Grid 布局，通过额外的 DOM 元素叠加来绘制加粗/加深的线**。

**最终方案：保留 CSS Grid + 用伪元素/额外 div 叠加中心线和边框**

具体做法：
1. 在 `.pixel-art-canvas` 容器上用 `::before` 和 `::after` 伪元素绘制中心十字线
2. 给 `.pixel-art-canvas` 的 `border` 加深颜色（外侧框线）
3. 中心线用绝对定位的 div 叠加（因为伪元素只有 2 个，不够用，需要额外 div）

**更简洁的方案：在 renderCanvas() 中为特定像素添加 CSS 类**

- 给处于中心行/列的 `.pixel` 元素添加更深的边框
- 给处于边缘的 `.pixel` 元素添加更深的边框
- 用 CSS 类控制边框颜色

**最终采用方案：在 renderCanvas() 中动态添加网格线叠加层**

在 `.pixel-art-canvas-container` 中叠加一个绝对定位的 SVG 或 div 层，专门绘制加深的中心线和边框。这样不影响现有像素交互。

具体步骤：

1. **修改 `renderCanvas()`**：在画布容器内追加一个绝对定位的叠加层，绘制：
   - 中心垂直线（x = canvasSize/2 的位置）
   - 中心水平线（y = canvasSize/2 的位置）
   - 四周边框（比现有 border 更深）

2. **修改 CSS**：
   - `.pixel-art-canvas-container` 添加 `position: relative`
   - 叠加层 `position: absolute; pointer-events: none;` 确保不影响交互
   - `.pixel-art-canvas` 的 `border` 颜色加深

3. **网格线颜色**：使用 `var(--color-text-hint)`（亮色 `#B8ADA3`，暗色也有对应值），比 `var(--color-divider)` 深但不会太突兀

## 二、透明色支持

### 现状分析

- 像素默认颜色为 `#FFFFFF`（白色）
- 橡皮擦将像素设为 `#FFFFFF`
- 导出时 `ctx.fillRect()` 绘制所有像素，无透明区域
- `ClearCanvasCommand.execute()` 使用 `this.pixels.fill('#FFFFFF')`

### 实现方案

用 `'transparent'` 字符串替代 `'#FFFFFF'` 作为"空"像素的值。

#### 需要修改的位置

1. **构造函数**（第 179 行）：`Array(...).fill('#FFFFFF')` → `Array(...).fill('transparent')`
2. **ClearCanvasCommand.execute()**（第 82 行）：`this.pixels.fill('#FFFFFF')` → `this.pixels.fill('transparent')`
3. **handlePixelClick - eraser**（第 513 行）：`if (oldColor !== '#FFFFFF')` 和 `new DrawPixelCommand(..., '#FFFFFF')` → 改为 `'transparent'`
4. **resizeCanvas()**（第 589 行）：`Array(size * size).fill('#FFFFFF')` → `Array(size * size).fill('transparent')`
5. **renderCanvas()**（第 301 行）：`style="background-color: ${color}"` → 透明像素需要特殊处理，显示棋盘格背景表示透明
6. **export()**（第 609-637 行）：透明像素不绘制（跳过 `fillRect`），PNG 自然保持透明
7. **CSS `.pixel`**（第 1106-1111 行）：移除 `background-color: #FFFFFF`，改为棋盘格背景表示透明

#### 透明像素的视觉表示

使用经典的棋盘格（checkerboard）背景表示透明区域，这是图像编辑器的通用做法：

```css
.pixel {
    background-image: 
        linear-gradient(45deg, #e0e0e0 25%, transparent 25%),
        linear-gradient(-45deg, #e0e0e0 25%, transparent 25%),
        linear-gradient(45deg, transparent 75%, #e0e0e0 75%),
        linear-gradient(-45deg, transparent 75%, #e0e0e0 75%);
    background-size: 8px 8px;
    background-position: 0 0, 0 4px, 4px -4px, -4px 0px;
}
```

当像素有颜色时，`background-color` 会覆盖棋盘格；当像素为 `transparent` 时，棋盘格显示出来。

但更简洁的做法是：在 `.pixel-art-canvas` 容器上设置棋盘格背景，透明像素不设置 `background-color`，让容器棋盘格透出来。不过由于 CSS Grid `gap` 的存在，容器背景已经是网格线颜色了。

**最终方案**：在 `renderCanvas()` 中，对透明像素设置一个 CSS 类 `.pixel-transparent`，该类使用棋盘格背景；非透明像素正常设置 `background-color`。

#### 导出时的透明处理

```javascript
for (let i = 0; i < this.pixels.length; i++) {
    if (this.pixels[i] === 'transparent') continue;  // 跳过透明像素
    const x = (i % this.canvasSize) * pixelSize;
    const y = Math.floor(i / this.canvasSize) * pixelSize;
    ctx.fillStyle = this.pixels[i];
    ctx.fillRect(x, y, pixelSize, pixelSize);
}
```

Canvas 默认背景就是透明的，不绘制的区域自然为透明，导出 PNG 时透明区域保持透明。

## 四、详细实现步骤

### 步骤 1：修改 CSS（components.css）

1. `.pixel-art-canvas-container` 添加 `position: relative`
2. `.pixel-art-canvas` 的 `border` 颜色从 `var(--color-divider)` 改为 `var(--color-text-hint)`
3. `.pixel` 移除 `background-color: #FFFFFF`
4. 新增 `.pixel-transparent` 类，使用棋盘格背景
5. 新增 `.pixel-grid-overlay` 绝对定位叠加层样式

### 步骤 2：修改 pixel_art.js - 透明色支持

1. 构造函数：`fill('#FFFFFF')` → `fill('transparent')`
2. `ClearCanvasCommand.execute()`：`fill('#FFFFFF')` → `fill('transparent')`
3. `handlePixelClick - eraser`：`'#FFFFFF'` → `'transparent'`
4. `resizeCanvas()`：`fill('#FFFFFF')` → `fill('transparent')`

### 步骤 3：修改 renderCanvas() - 透明像素 + 网格线叠加

1. 透明像素添加 `pixel-transparent` 类，非透明像素设置 `background-color`
2. 在画布容器中追加网格线叠加层 div，绘制中心线和加深边框

### 步骤 4：修改 export() - 透明导出

1. 跳过 `transparent` 像素，不调用 `fillRect`
2. Canvas 默认透明背景，PNG 导出自然支持透明

### 步骤 5：更新 db.json 版本号

`3.1.4` → `3.1.5`
