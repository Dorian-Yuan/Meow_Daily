# 四款 Meow Phone 应用全面优化计划

## 优化原则（从已优化三款应用中提炼）

1. **游戏化深度**：从单次交互升级为有目标、有挑战、有反馈的完整游戏循环（参考 yarn\_ball 从纯解压→挑战模式）
2. **难度/模式选择**：提供可配置的难度或模式选项，增加重玩价值（参考 cat\_sweep 三难度+自定义）
3. **设置集成**：在 Meow Phone 设置页面增加偏好配置（cat\_sweep/pixel\_art/yarn\_ball 均有）
4. **CSS 变量体系**：统一使用 `var(--color-*)` 变量，移除硬编码颜色
5. **结果反馈升级**：采用全屏遮罩+居中卡片+弹出动画的统一风格（参考 cat\_sweep/yarn\_ball 的结果弹窗）
6. **数据持久化**：最佳成绩/历史记录保存到 localStorage（参考 pixel\_art 的缓存机制）
7. **代码架构**：清晰的闭包状态管理、destroy 资源清理、公共 API 暴露

***

## 一、猫咪占卜 (cat\_fortune.js) 优化

### 当前问题

* 运势文本太少（每级仅3条，共15条），重复率高

* "再摇一次"只换同等级文字不换运势等级，按钮文案误导

* 无历史记录，缺乏连续性和仪式感

* 动画效果单一，缺乏惊喜感

* 无设置项

### 优化内容

#### 1.1 大幅扩充运势文本库

* 每级运势从 3 条扩充到至少 10 条（共 50+ 条），降低重复感

* 文本风格保持可爱调皮的猫咪语气

#### 1.2 "再摇一次"改为"换签文"

* 按钮文案从 `再摇一次` 改为 `换签文`，准确反映功能（只换同等级文字）

* 同时增加 `重新求签` 按钮，允许完全重新抽签（新等级+新文字）

#### 1.3 增加运势历史记录

* 使用 localStorage 存储最近 7 天的运势记录（key: `meow_fortune_history`）

* 在占卜结果下方增加"查看历史"入口，展示最近 7 天的运势等级+emoji

* 历史记录使用横向滚动的小卡片展示

#### 1.4 增强动画效果

* 思考动画：增加猫咪emoji旋转+星星粒子效果

* 揭示动画：运势等级文字从模糊到清晰的渐变效果（`filter: blur` → 清晰）

* 大吉/凶特殊效果：大吉时金色光芒闪烁，凶时灰色抖动

#### 1.5 CSS 变量统一

* 确保所有样式使用 `var(--color-*)` 变量（当前 fortune-level/fortune-text 已使用，基本OK）

## 二、猫咪翻牌 (cat\_memory.js) 优化

### 当前问题

* 固定 4x4 难度，无选择

* emoji 种类固定，缺乏变化

* 无最佳成绩记录

* 星级评价仅按步数，不考虑时间

* style 标签重复追加

* 结果覆盖层无法关闭查看棋盘

* 无设置项

* CSS 大量硬编码颜色

### 优化内容

#### 2.1 增加难度选择

* 三种难度：

  * 简单：3x4=12张（6对）

  * 中等：4x4=16张（8对，当前默认）

  * 困难：5x4=20张（10对）

* 游戏开始前显示难度选择界面（类似 whack\_mouse 的开始界面）

* 根据难度动态调整 grid-template-columns

#### 2.2 增加 emoji 主题切换

* 3 套 emoji 主题：

  * 猫咪：`['🐱', '😺', '😸', '😹', '😻', '😼', '🐟', '🧶', '🐾', '🐭']`

  * 食物：`['🍣', '🍕', '🍔', '🌮', '🍜', '🍰', '🍩', '🍪', '🧁', '🍿']`

  * 自然：`['🌸', '🌺', '🌻', '🌹', '🍀', '🌈', '⭐', '🌙', '☀️', '❄️']`

* 每局随机选择或从设置中指定主题

#### 2.3 增加最佳成绩记录

* 使用 localStorage 存储每个难度的最佳成绩（最少步数+最短时间）

* key: `meow_memory_best_{difficulty}`

* 结果弹窗中显示"新纪录！"标识（如果打破记录）

#### 2.4 改进星级评价

* 综合步数和时间双维度评级：

  * 简单：≤8步且≤20秒→三星，≤12步→两星，否则一星

  * 中等：≤12步且≤40秒→三星，≤18步→两星，否则一星

  * 困难：≤16步且≤60秒→三星，≤24步→两星，否则一星

#### 2.5 结果弹窗升级

* 采用统一的全屏遮罩+居中卡片+弹出动画风格（与 cat\_sweep/yarn\_ball 一致）

* 点击遮罩可关闭查看棋盘

* 显示：星级 + 步数 + 用时 + 最佳成绩对比

#### 2.6 CSS 变量统一 + style 标签管理

* 移除内联 style 标签，将样式迁移到 components.css

* 所有硬编码颜色替换为 `var(--color-*)` 变量

  * `#ffb6c1/#ffd1dc` → `var(--color-primary)` 渐变

  * `#a8e6cf/#dcedc1` → `var(--color-bg-warm)` 渐变

  * `#555` → `var(--color-text-main)`

  * `#f0f0f0` → `var(--color-divider)`

#### 2.7 增加设置项

* Meow Phone 设置页面增加猫咪翻牌配置：

  * 默认难度：简单/中等/困难

  * emoji 主题：猫咪/食物/自然/随机

* store.js DEFAULT\_DB 中增加 `cat_memory: { difficulty: 'medium', theme: 'random' }`

## 三、打地鼠 (whack\_mouse.js) 优化

### 当前问题

* 固定 30 秒/9 宫格，无难度选择

* 无最高分记录

* 无连击/combo 系统

* 地鼠种类单一

* 无倒计时警告

* CSS 大量硬编码颜色

* 结果界面硬编码 `color:#fff`

* style 标签重复追加

* 无设置项

### 优化内容

#### 3.1 增加难度选择

* 三种难度：

  * 简单：30秒，地鼠显示时间较长，无双鼠

  * 中等：30秒，当前默认难度递增

  * 困难：45秒，地鼠显示时间更短，10秒后双鼠+炸弹鼠

* 开始界面增加难度选择按钮

#### 3.2 增加 Combo 连击系统

* 连续击中地鼠（2秒内再次击中）累加 combo 计数

* combo 加分：基础 10 分 × combo 倍率（2连=1.5x, 3连=2x, 4连=2.5x, 5+连=3x）

* combo 显示：在 HUD 区域显示当前 combo 数和倍率

* combo 断连时显示"连击中断"提示

* 击中时屏幕轻微震动效果（CSS transform）

#### 3.3 增加特殊地鼠

* 🐭 普通鼠：+10 分

* 🐹 金色鼠：+30 分，出现概率 15%，显示时间更短

* 💣 炸弹鼠：-20 分，点击扣分，仅困难模式出现，概率 10%

* 特殊地鼠有不同视觉样式（金色鼠发光，炸弹鼠红色边框）

#### 3.4 增加最高分记录

* localStorage 存储每个难度的最高分

* key: `meow_whack_best_{difficulty}`

* 结果弹窗中显示最高分对比

#### 3.5 倒计时警告

* 最后 5 秒：时间数字变红 + 闪烁动画

* 最后 3 秒：每秒震动提示

#### 3.6 结果弹窗升级

* 采用统一的全屏遮罩+居中卡片+弹出动画风格

* 显示：分数 + combo 最高连击 + 爪印评级 + 最高分对比

* 移除所有内联 `style="color:#fff"` 硬编码

#### 3.7 CSS 变量统一 + style 标签管理

* 移除内联 style 标签，将样式迁移到 components.css

* 硬编码颜色替换：

  * `#ff6b6b/#ee5a24` → `var(--color-primary)` 渐变

  * `#ffd700` → 保持金色（评分专用）

  * `#fff` → `var(--color-text-on-primary)`

  * `#ccc` → `var(--color-text-hint)`

  * 地洞颜色保持棕色系（游戏主题色，可接受）

#### 3.8 增加设置项

* Meow Phone 设置页面增加打地鼠配置：

  * 默认难度：简单/中等/困难

* store.js DEFAULT\_DB 中增加 `whack_mouse: { difficulty: 'medium' }`

## 四、喵喵琴 (meow\_piano.js) 优化

### 当前问题

* 音色单一（正弦波，像蜂鸣器）

* 只有一个八度（C4-C5），音域极窄

* 无预设曲目/教程

* 录制功能体验差（无取消、无进度、不持久化）

* 琴键尺寸固定，小屏可能溢出

* 无黑键

* CSS 硬编码颜色

* 无键盘快捷键

* 无设置项

### 优化内容

#### 4.1 改善音色

* 从纯正弦波改为更丰富的合成音色：

  * 使用双振荡器叠加：基频(sine) + 泛音(triangle, 频率×2, 音量0.15)

  * 添加轻微颤音（vibrato）：用低频振荡器(LFO)调制主频率

  * ADSR 包络优化：快速起音(5ms) → 短衰减(100ms) → 持续(0.15) → 释放(300ms)

* 效果：从"电子蜂鸣器"变为"柔和的猫咪风铃"音色

#### 4.2 增加八度切换

* 支持两个八度范围：C4-C5（当前）和 C5-C6

* 底部增加八度切换按钮：`◀ 低音` / `高音 ▶`

* 切换时更新 FREQS 数组和琴键显示的音符名

#### 4.3 增加预设曲目

* 内置 3 首简单示范曲目（用音符索引+时间戳表示）：

  * 《小星星》：C C G G A A G - F F E E D D C

  * 《生日快乐》：C C D C F E - C C D C G F

  * 《两只老虎》：C D E C - C D E C - E F G - E F G

* 增加"示范曲目"按钮，点击后展示曲目列表

* 选择曲目后自动播放并高亮对应琴键（教学功能）

#### 4.4 录制功能增强

* 录制时显示进度条（30秒限制，显示剩余时间）

* 录制中可点击停止按钮提前结束

* 播放时琴键同步高亮

* 录制数据保存到 localStorage（最近 1 条），退出不丢失

* key: `meow_piano_recording`

#### 4.5 琴键响应式布局

* 琴键宽度从固定 38px 改为 `flex: 1; min-width: 0`

* 高度从固定 120px 改为响应式

* 确保在 320px 宽度下 8 个键仍可正常使用

#### 4.6 键盘快捷键支持

* A S D F G H J K 对应 8 个琴键

* 在琴键上显示对应的键盘字母（仅桌面端显示，移动端隐藏）

* 通过 `window.matchMedia('(hover: hover)')` 检测桌面端

#### 4.7 CSS 变量统一 + style 标签管理

* 移除内联 style 标签，将样式迁移到 components.css

* 硬编码颜色替换：

  * `#374151` → `var(--color-card-bg)`

  * `#4B5563` → `var(--color-text-sub)`

  * `#fff` → `var(--color-text-on-primary)`

  * `#9CA3AF` → `var(--color-text-hint)`

  * `#EF4444` → 保持（录制状态红色，语义明确）

#### 4.8 增加设置项

* Meow Phone 设置页面增加喵喵琴配置：

  * 音色：柔和/明亮/电子（影响振荡器类型和泛音配置）

* store.js DEFAULT\_DB 中增加 `meow_piano: { tone: 'soft' }`
